---
title: Share a Public Collection
---

Use this workflow when you want guests to browse a public set of links without logging in.

## Steps

1. Decide whether the collection should be organized by a list, a tag, or both.
2. Set the relevant links to **Public**.
3. Set the list or tag to **Public**.
4. Enable guest access in **System Settings**.
5. Open the guest view in a private browser window to verify what visitors can see.
6. Share the guest link, list, tag, or RSS feed URL.

## Visibility Rules

Guest users can only see public content when guest access is enabled.

- Public links can be visible to guests.
- Internal links are visible only to logged-in users.
- Private links are visible only to their owner.

The same model applies to lists and tags. If a public list is missing links, check the visibility of the links inside it.

## RSS Feeds

Public RSS feeds are available for:

- all public guest links,
- public links in a public list,
- public links with a public tag.

See [RSS Feeds]({{< relref path="docs/v2/application/rss-feeds.md" >}}) for feed URLs.
